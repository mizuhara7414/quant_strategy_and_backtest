# HMM 輪動策略調校紀錄

## 最終策略檔案
`hmm_rotation.py` — 0050.TW ↔ 00632R.TW 雙標的輪動

---

## 起點：基礎 HMM 狀態偵測

**檔案**：`hmm_regime.py`
**標的**：^TWII（台灣加權指數）
**特徵**：`ret_1d`, `vol_5d`, `mom_10d`, `vol_ratio`
**邏輯**：bull → 100% 持有 / sideways → 動態倉位 / bear → 0%（空倉）

---

## 調校階段一：特徵改進

**問題**：sideways 期間累積報酬 65%，但 bull 佔比只有 17.6%，bull 狀態辨識率偏低
**改動**：將 `vol_5d` 替換為 `mom_20d`
**結果**：bull 佔比從 17.6% 提升至 37.4%，sideways 辨識改善
**教訓**：波動率特徵對 bull/sideways 區分貢獻有限；動能特徵更能捕捉趨勢方向

---

## 調校階段二：標的切換

**問題**：^TWII 是指數，無法實際交易
**改動**：標的換為 `0050.TW`（實際可交易的台灣50 ETF）
**影響**：資料略有差異，但策略邏輯不變

---

## 調校階段三：槓桿 ETF 實驗（失敗）

**嘗試**：bull 時使用 `00631L.TW`（元大台灣50正2）追求超額報酬
**結果**：MDD -60.77%（遠差於 B&H -55.14%），Sharpe 0.838 < B&H 1.002
**原因**：槓桿 ETF 波動衰變（Volatility Decay）在震盪市場中嚴重侵蝕報酬
**結論**：放棄槓桿 ETF，回到 0050

---

## 調校階段四：SIDEWAYS_POS 參數優化

**工具**：Grid Search（64 種組合）→ Differential Evolution（連續搜尋空間）
**問題**：DE 優化在測試集上 Sharpe 1.698，但真正驗證集上只有 0.867，差距 0.831
**結論**：嚴重過擬合，放棄 DE 參數，回到邏輯選取的參數：

```python
SIDEWAYS_POS = {"both": 1.00, "either": 0.70, "neither": 0.20}
```

**教訓**：`log-likelihood ≠ 交易績效`；DE 優化容易對測試集過擬合

---

## 調校階段五：加入交易成本

**改動**：
```python
FEE_RATE = 0.001425   # 買賣手續費 0.1425%
TAX_RATE = 0.001      # 賣出證交稅 0.1%
```

**影響**：頻繁交易的版本報酬大幅衰減，促使後續改為週頻調倉

---

## 調校階段六：確認期機制（Confirmation Period）

**問題**：HMM 狀態每天都可能切換，造成頻繁交易和手續費拖累
**改動**：加入 `CONFIRM_DAYS = 2`，連續 N 天同一狀態才確認切換
**結果**：交易次數減少，Sharpe 改善

---

## 調校階段七：週頻調倉

**改動**：只在每週五執行倉位調整（`dates[i].dayofweek != 4` 則跳過）
**結果**：報酬提升，交易成本降低
**狀態改變次數**：95 次 HMM 狀態變化 → 實際交易 62 次

---

## 調校階段八：多標的輪動策略

**動機**：純空倉（bear=0%）讓資金閒置，想讓資金在熊市繼續工作
**嘗試 1**：`00679B.TW`（美債 ETF）→ Yahoo Finance 找不到，放棄
**嘗試 2**：反向 ETF `00632R.TW`（元大台灣50反1）→ 採用
**邏輯**：bull/sideways → 0050.TW / bear → 00632R.TW

**初始結果（無雙重確認）**：
```
總報酬率：30.34%（B&H 165.42%）
Sharpe：0.345（B&H 1.172）
MDD：-23.99%
測試集熊市佔比：44.7%（過高）
```

**問題**：HMM 誤判熊市時持有反向 ETF = 主動虧損，誤判代價遠高於純空倉

---

## 調校階段九：雙重技術確認

**改動**：HMM 判熊還需同時滿足技術條件才算真熊市，否則降級為 sideways

```python
is_technical_bear = (eq_price < ma60[i]) and (mom[i] < 0)
if label == "bear" and not is_technical_bear:
    label = "sideways"
```

**結果**：
```
總報酬率：165.02%
Sharpe：1.261
MDD：-22.05%
```

**成效**：與 B&H 報酬相當，但 Sharpe 更高、MDD 更低

---

## 調校階段十：非對稱確認期（失敗）

**嘗試**：進熊快（1天）、出熊慢（3天）
```python
CONFIRM_DAYS_ENTER = 1
CONFIRM_DAYS_EXIT  = 3
```

**結果**：
```
總報酬率：148.79%（下降）
Sharpe：1.171（下降）
MDD：-22.05%（無改善）
```

**原因**：進熊更快在多頭期間誤切反向 ETF；出熊更慢錯過反彈 → 雙重損失
**結論**：放棄非對稱確認，還原 `CONFIRM_DAYS = 2`

---

## 調校階段十一：高水位停損守衛（災難性失敗）

**嘗試**：組合從高點回撤超過 8% 就強制切換防禦模式
```python
DRAWDOWN_GUARD = 0.08
if (total - peak_val) / peak_val < -DRAWDOWN_GUARD:
    label = "bear"
```

**結果**：
```
總報酬率：133.12%
Sharpe：0.420
MDD：-54.67%（遠差於 B&H）
實際交易：14 次
```

**原因**：進入反向 ETF 後，市場回升 → 反向 ETF 虧損 → 組合下跌 → 守門持續觸發 → 永久鎖在反向 ETF，形成無法自行退出的死循環。途中反向 ETF 在某次真正下跌中暴漲（帳面飆至 115），但最終仍崩潰
**教訓**：高水位停損 + 反向 ETF 組合在沒有出口條件時極度危險

---

## 調校階段十二：部分倉位 + 降低 sideways 暴露

**改動**：
```python
BEAR_INVERSE_PCT = 0.50   # 熊市只押 50% 反向 ETF
SIDEWAYS_POS = {"both": 1.00, "either": 0.60, "neither": 0.10}
```

**結果**：
```
總報酬率：148.66%
Sharpe：1.413
MDD：-18.13%
```

---

## 調校階段十三：更嚴格的技術門檻（失敗）

**嘗試**：`mom[i] < -0.01`（動能跌超過 1% 才算確認）
**結果**：MDD -31.42%（比 B&H 還差）
**原因**：過濾太嚴，真熊市也被錯放，2022 年大跌時仍持股 → MDD 上升
**結論**：維持 `mom[i] < 0`

---

## 調校階段十四：調低反向 ETF 比例

**改動**：`BEAR_INVERSE_PCT = 0.30`（50% → 30%）
**結果**：
```
總報酬率：146.20%
Sharpe：1.471
MDD：-16.68%
```

---

## 調校階段十五：還原 sideways 倉位（最終版本）

**觀察**：sideways 倉位降低是報酬壓縮的主因，而 sideways 期間市場未必在跌
**改動**：sideways 倉位還原至原始值
```python
SIDEWAYS_POS = {"both": 1.00, "either": 0.70, "neither": 0.20}
BEAR_INVERSE_PCT = 0.30
```

**最終結果**：
```
總報酬率：156.42%（B&H 165.42%）
Sharpe：1.503（B&H 1.172）
MDD：-17.00%（B&H -27.48%）
實際交易次數：68
```

## 最終策略參數

```python
EQUITY_SYMBOL    = "0050.TW"
INVERSE_SYMBOL   = "00632R.TW"
PERIOD           = "10y"
N_STATES         = 3
N_ITER           = 300
TRAIN_RATIO      = 0.60
CASH             = 100_000
CONFIRM_DAYS     = 2
FEE_RATE         = 0.001425
TAX_RATE         = 0.001
BEAR_INVERSE_PCT = 0.30
SIDEWAYS_POS     = {"both": 1.00, "either": 0.70, "neither": 0.20}
FEATURE_COLS     = ["ret_1d", "mom_10d", "mom_20d", "vol_ratio"]
```

### 核心規則
1. **HMM** 偵測市場狀態（bull / sideways / bear）
2. **雙重確認**：bear 需同時滿足 跌破MA60 + 20日動能 < 0
3. **週頻調倉**：每週五執行，倉位變化 < 5% 不動
4. **bull** → 100% 0050
5. **sideways** → 動態倉位（MA60 + 動能條件決定 100% / 70% / 20%）
6. **bear** → 30% 00632R + 70% 現金

---

## 策略適用環境

**最有利**：急跌急漲的 V 型反轉、多空輪替明顯、熊市持續 3–12 個月
**最不利**：慢熊（如 2000–2003）、長期橫盤震盪、長多頭中的頻繁假熊信號

---

## 已知限制

- 00632R.TW 上市時間限制（~2014年後），無法測試更早的歷史
- 僅單次 train/test 切分，缺乏 Walk-Forward 驗證
- HMM 熊市判斷佔比 44.7%，仍偏高
- 未考慮滑價與買賣價差
